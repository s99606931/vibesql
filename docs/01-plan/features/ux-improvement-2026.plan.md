## Executive Summary

| Perspective | Content |
|-------------|---------|
| **Problem** | 전체 메뉴에 걸쳐 `confirm()`/`prompt()` 남용, 빈 상태 CTA 부재, 대시보드 상세 뷰 미구현 등 실사용 시 마찰이 존재함 |
| **Solution** | 고임팩트·저난이도 항목부터 순차 개선: UI 패턴 정규화 → 핵심 기능 완성 → 데이터 탐색 UX 강화 |

### [홈] /home

**지원 기능 목록** (코드에서 확인된 기능들):
- ✅ 3단계 온보딩 워크플로 (연결 → AI 설정 → 워크스페이스)
- ✅ DB 연결/AI 프로바이더 상태 실시간 체크 (React Query)
- ✅ 진행 상태 요약 배너 (완료 시 초록 → 미완료 시 경고)
- ✅ 다음 액션 CTA 동적 변경
- ✅ 사용 통계 4종 (전체 쿼리, 성공률, 저장된 쿼리, 평균 실행시간)
- ✅ 최근 쿼리 3개 미리보기
- ✅ 전체 메뉴 아코디언 사용 가이드
- ✅ 명령 팔레트 안내 (⌘K)

**사용자 경험 이슈** (실사용 시뮬레이션):
- ⚠️ 최근 쿼리 항목 클릭 시 아무 일도 안 일어남 — 클릭 커서가 있지만 워크스페이스로 이동하는 핸들러가 없음
- ⚠️ 사용 통계 카드가 로딩 중일 때 "—"만 표시 — Skeleton 없이 숫자가 갑자기 나타나 레이아웃 튐 현상
- ❌ 통계 에러 상태 처리 없음 — API 실패 시 영원히 "—"로 남음
- ⚠️ 메뉴 가이드가 9개 아코디언 — 페이지 로드 시 모두 닫혀 있어 어디서부터 시작해야 할지 모름

**개선 제안**:
1. 최근 쿼리 항목에 클릭 → 워크스페이스 이동 핸들러 추가 (난이도: 쉬움)
2. 통계 카드에 Skeleton 로딩 적용 (난이도: 쉬움)
3. 첫 번째 미완료 단계 가이드 아코디언 자동 열기 (난이도: 쉬움)

### [워크스페이스] /workspace

**지원 기능 목록** (코드에서 확인된 기능들):
- ✅ 자연어 → SQL 생성 (⌘Enter 단축키)
- ✅ DB 연결 선택 드롭다운 (TopBar)
- ✅ SQL 에디터 (CodeMirror, 인라인 편집)
- ✅ 결과 테이블 / 차트 / SQL 설명 탭
- ✅ CSV 내보내기
- ✅ 저장 / 공유 버튼 (결과 있을 때)
- ✅ 대시보드 추가 버튼
- ✅ 템플릿 불러오기 모달
- ✅ AI 신뢰도 표시 (high/medium/low)
- ✅ 예시 질문 칩
- ✅ 수정됨 경고 Pill
- ✅ URL 파라미터로 SQL 프리로드 (?sql=...)
- ✅ 스키마 + 용어집 병렬 로드 (AI 컨텍스트)
- ✅ 히스토리 자동 기록

**사용자 경험 이슈** (실사용 시뮬레이션):
- ⚠️ 대시보드 추가 시 `prompt()`/`alert()` 사용 — 브라우저 기본 다이얼로그로 현대 UI 기대에 어긋남
- ⚠️ 연결이 없을 때 SQL 생성 시도하면 "No schema provided" 경고 없이 진행 — 분석가가 왜 결과가 이상한지 모름
- ⚠️ 저장 시 폴더 선택 없이 무조건 "기본" 폴더 — 분류 체계 없이 쌓임
- ❌ 결과가 1,000행 제한인데 사용자에게 고지 없음 — "왜 데이터가 적지?" 의문 발생
- ⚠️ 차트 탭에서 차트 유형 변경 불가 (ResultChart 컴포넌트 내부에서 자동 결정)

**개선 제안**:
1. 대시보드 추가 드롭다운을 인라인 Popover로 교체, alert 제거 (난이도: 중간)
2. 결과 테이블 헤더에 "1,000행 제한" 배지 표시 (난이도: 쉬움)
3. 저장 시 폴더 선택 드롭다운 추가 (난이도: 쉬움)
4. 연결 미선택 상태에서 생성 시도 시 인라인 경고 표시 (난이도: 쉬움)

### [템플릿] /templates

**지원 기능 목록** (코드에서 확인된 기능들):
- ✅ 카테고리 필터 (전체/분석/운영/보고서/디버깅/내 템플릿)
- ✅ 텍스트 검색
- ✅ SQL 미리보기 토글
- ✅ 템플릿 저장 모달 (이름/설명/카테고리/방언/태그)
- ✅ 삭제 (내 템플릿만)
- ✅ "사용" 버튼 → 워크스페이스로 이동
- ✅ 통계 카드 (전체/기본제공/내 템플릿 수)

**사용자 경험 이슈** (실사용 시뮬레이션):
- ⚠️ 삭제 시 `confirm()` 사용 — 브라우저 기본 다이얼로그
- ⚠️ 템플릿 저장 시 워크스페이스 현재 SQL을 자동으로 채워주지 않음 — 복사-붙여넣기 필요
- ❌ 빈 상태에서 템플릿 추가 유도 CTA가 없음 (내 템플릿 탭 선택 시)
- ⚠️ SQL 보기 토글이 ▶/▼ 텍스트 버튼 — 시각적으로 약함

**개선 제안**:
1. confirm() → 인라인 확인 버튼(Undo 패턴) 또는 모달로 교체 (난이도: 쉬움)
2. 워크스페이스에서 "현재 SQL로 템플릿 저장" 시 자동 pre-fill (난이도: 쉬움)
3. 카테고리별 빈 상태 CTA 추가 (난이도: 쉬움)

### [히스토리] /history

**지원 기능 목록** (코드에서 확인된 기능들):
- ✅ 날짜 그룹핑 (오늘/어제/이전)
- ✅ 필터 (전체/성공/실패/즐겨찾기)
- ✅ 텍스트 검색
- ✅ 재실행 버튼 (워크스페이스로 이동)
- ✅ 별표(즐겨찾기) 토글
- ✅ 삭제 버튼
- ✅ "더 보기" 로드 (50개씩 추가)
- ✅ Skeleton 로딩

**사용자 경험 이슈** (실사용 시뮬레이션):
- ⚠️ 삭제 시 `confirm()` 사용
- ⚠️ hover 시에만 액션 버튼 표시 — 터치 디바이스에서 접근 불가
- ⚠️ "실패" 필터가 `FAILURE` 파라미터를 보내지만 API는 `ERROR` 상태를 사용 — 필터가 실제로 동작하지 않을 수 있음
- ❌ 실패한 쿼리의 에러 메시지가 목록에 표시되지 않음 — "왜 실패했는지" 알 수 없음
- ⚠️ 히스토리 일괄 삭제 기능 없음 — 오래된 기록 정리 불가

**개선 제안**:
1. 필터 버그 수정: "실패" 필터의 statusParam을 `ERROR`로 변경 (난이도: 쉬움)
2. 실패 항목 행 클릭 시 에러 메시지 인라인 표시 (난이도: 쉬움)
3. confirm() → 인라인 삭제 확인 UX 교체 (난이도: 쉬움)
4. 일괄 삭제 체크박스 + 선택 삭제 (난이도: 중간)

**지원 기능 목록** (코드에서 확인된 기능들):
- ✅ 폴더별 그룹핑
- ✅ 텍스트 검색
- ✅ 워크스페이스에서 열기 / 즉시 실행 / 스케줄 등록
- ✅ 이름 편집 (prompt 사용)
- ✅ 삭제 (confirm 사용)
- ✅ 버전 히스토리 패널 (우측 슬라이드)
- ✅ 새 폴더 생성 (prompt 사용)
- ✅ 태그 표시

**사용자 경험 이슈** (실사용 시뮬레이션):
- ⚠️ 새 폴더 생성, 이름 편집 모두 `prompt()` 사용 — 3곳에서 브라우저 기본 다이얼로그
- ⚠️ 새 폴더 생성 로직 버그: 미분류 쿼리를 신규 폴더로 강제 이동시킴 — 기대 동작과 다름
- ❌ 폴더 이름 변경 기능 없음
- ⚠️ 쿼리를 다른 폴더로 이동하는 UI 없음 (drag-and-drop 또는 이동 버튼)
- ⚠️ hover 시에만 액션 버튼 노출 — 모바일에서 접근 불가

**개선 제안**:
1. prompt() 전부 인라인 편집(inline edit) 또는 소형 모달로 교체 (난이도: 중간)
2. 폴더 이름 변경 기능 추가 (난이도: 쉬움)
3. 쿼리 이동 드롭다운 (폴더 선택) 추가 (난이도: 중간)
4. 새 폴더 생성 로직 수정: 빈 폴더를 UI에만 생성하고 쿼리 이동은 별도 (난이도: 쉬움)

### [스케줄] /schedules

**지원 기능 목록** (코드에서 확인된 기능들):

- ✅ 마지막 실행 시간 및 결과 표시
- ✅ 빈 상태 처리 (Calendar 아이콘 + CTA)
- ✅ 자동 실행 워커 미구현 안내 배너

**사용자 경험 이슈** (실사용 시뮬레이션):
- ⚠️ 삭제 시 `confirm()` 사용
- ❌ 활성/비활성 토글 스위치 없음 — 편집 모달 열어야만 변경 가능

**개선 제안**:
1. 목록에서 활성/비활성 토글 스위치 직접 노출 (난이도: 쉬움)
2. Cron 표현식 입력 시 "다음 실행: X월 X일 오전 X시" 프리뷰 표시 (난이도: 중간)
3. 연결 선택 드롭다운 스케줄 폼에 추가 (난이도: 쉬움)
4. confirm() → 인라인 확인 교체 (난이도: 쉬움)

- ✅ SSL 설정

**사용자 경험 이슈** (실사용 시뮬레이션):
- ⚠️ 삭제 시 `confirm()` 사용

**개선 제안**:
1. 테이블에 database 컬럼 추가, 테스트 실패 시 에러 메시지 툴팁 표시 (난이도: 쉬움)
2. 현재 활성 연결 강조 표시 (Pill "활성" 배지) (난이도: 쉬움)
3. 편집 폼을 모달로 이전 (난이도: 중간)

### [용어 사전] /glossary

**지원 기능 목록** (코드에서 확인된 기능들):
- ✅ 마스터-디테일 레이아웃 (왼쪽 목록 + 오른쪽 상세)
- ✅ 검색 필터
- ✅ 용어 추가 (모달 폼)
- ✅ 카테고리 분류 (매출/사용자/지표/기타)
- ✅ SQL 힌트 필드
- ✅ 삭제 버튼
- ✅ AI 활용 안내 callout

**사용자 경험 이슈** (실사용 시뮬레이션):
- ❌ 용어 편집 기능 없음 — 삭제 후 재등록만 가능
- ⚠️ 카테고리가 하드코딩 4종 — 팀마다 다른 분류 체계 반영 불가
- ❌ 대량 import 기능 없음 — 기존 용어집 CSV/JSON에서 일괄 등록 불가

- ⚠️ 모바일에서 마스터-디테일 레이아웃이 깨짐 (280px 고정 왼쪽 패널)

## 1. Overview

### 1.1 Purpose

vibeSQL의 13개 메뉴를 데이터 분석가 페르소나로 전수 리뷰한 결과, 핵심 마찰 포인트를 식별하고 사용자 경험을 개선한다.

### 1.2 Background

전체 서비스는 기능적으로는 구현되어 있으나, (1) 브라우저 기본 다이얼로그 남용, (2) 편집 기능 누락, (3) 핵심 페이지 미구현, (4) 필터 버그 등 완성도 이슈가 발견되었다. 이를 우선순위화하여 순차 개선한다.

## 2. Scope

### 2.1 In Scope

- [ ] `confirm()` / `prompt()` / `alert()` → 인라인 모달 / toast 교체 (전체 서비스)
- [ ] 히스토리 "실패" 필터 버그 수정 (`ERROR` → `FAILURE` 파라미터 불일치)

- [ ] 스키마 페이지 연결 선택 드롭다운 추가
- [ ] 저장됨 새 폴더 로직 버그 수정
- [ ] 대시보드 생성 prompt() 교체
- [ ] 홈 최근 쿼리 클릭 핸들러 추가
- [ ] 스케줄 목록 활성/비활성 토글 직접 노출
