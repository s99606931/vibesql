## 1. 최초 설정

`docker compose up -d --build` 가 끝나면 <http://localhost:3000> 으로 접속하세요. `/signin` 으로 자동 이동합니다.

**처음 가입한 계정이 자동으로 ADMIN** 이 됩니다. 비밀번호 재설정 흐름이 아직 없으니 실제 이메일과 강력한 비밀번호를 사용하고 분실하지 마세요.

`VIBESQL_DEV_AUTH_BYPASS=1` 로 실행 중이면(개발 전용) 로그인 화면을 건너뛰고 모든 요청이 dev 사용자로 처리됩니다. **프로덕션에서는 절대 켜지 마세요.**

## 2. 로그인

- `/signin` 페이지에 **로그인** / **회원가입** 두 탭이 있습니다
- 이메일 + 비밀번호만 사용 (v1에서는 OAuth 미지원)
- 성공 시 `httpOnly` 쿠키 `vs-session` 이 설정되고 `/home` 으로 이동합니다

## 3. 데이터베이스 연결 추가

vibeSQL은 데모 데이터를 제공하지 않습니다 — 본인의 DB를 연결합니다.

1. 사이드바에서 **연결** 클릭
2. **새 연결** 버튼
3. 입력:
   - **이름** — 알아보기 쉬운 이름 (예: `prod-readonly`)
   - **종류** — PostgreSQL (MySQL/SQLite 지원 예정)
   - **호스트 / 포트 / 데이터베이스 / 사용자명 / 비밀번호**
   - **SSL** — 클라우드 DB(RDS, Supabase, Neon 등)는 활성화
4. **테스트** 버튼으로 연결 확인 — 서버 버전과 응답 지연이 표시됩니다
5. 저장 — 비밀번호는 `CONNECTION_ENCRYPTION_KEY` 로 AES-256-GCM 암호화되어 저장됩니다

> ⚠ **읽기 전용 DB 사용자를 사용하세요.** vibeSQL의 SQL 가드가 `SELECT` 외 모든 구문을 차단하지만, 다중 방어가 항상 더 안전합니다. 노출하려는 스키마에 대해서만 `SELECT` 권한을 가진 전용 PostgreSQL role을 만드세요.

## 4. 스키마 둘러보기

연결을 추가한 후 **스키마** 페이지에서 `information_schema` 와 `pg_class` 를 introspect한 결과를 볼 수 있습니다:

- `public` 스키마의 모든 테이블
- 대략적인 행 수 (`reltuples` 기반)
- 컬럼 목록
- **PII 배지** — `email`, `phone`, `ssn`, `password`, `card_number` 같은 패턴의 컬럼이 있는 테이블에 자동 표시. 어떤 데이터를 노출할지 결정할 때 참고하세요.

테이블 이름을 클릭하면 컬럼 상세를 볼 수 있고, 연결 셀렉터로 다른 데이터셋으로 전환할 수 있습니다.

## 5. 첫 자연어 질문

**워크스페이스** 로 이동하세요. 페이지는 세 영역으로 구성됩니다: 자연어 입력(상단), 생성된 SQL(중앙), 결과 테이블(하단).

1. 질문을 입력합니다. 한국어/영어 모두 가능하며, AI는 답변(설명)도 같은 언어로 작성합니다.

   *예시:*
   - "이번 주 주문이 가장 많은 상위 10명 사용자"
   - "결제 실패율이 가장 높은 방법은?"
   - "지난 6개월 카테고리별 월매출 추이"

2. **SQL 생성** 클릭. 다음이 일어납니다:
   - 활성 연결의 스키마 컨텍스트를 추출
   - NL + 스키마 + 방언을 Claude에 전송
   - 반환된 SQL을 AST 가드로 검증
   - 신뢰도 배지(높음/중간/낮음)와 함께 SQL을 에디터에 스트리밍

3. SQL을 검토합니다. 에디터는 편집 가능 — 필요하면 직접 수정하세요.

4. **실행** 클릭. 읽기 전용 — `SELECT` 만 실행되고 그 외 구문은 DB 도달 전에 차단됩니다.

## 6. 결과 다루기

결과 테이블은 다음을 지원합니다:

- **정렬** — 컬럼 헤더 클릭
- **검색** — 보이는 컬럼에 대한 텍스트 필터
- **페이지네이션** — 페이지당 50행, 큰 결과는 스트리밍
- **차트** — 컬럼 타입에 따라 라인/막대/파이 차트로 전환
- **내보내기** — 현재 결과를 CSV로 다운로드

모든 실행은 **히스토리** 에 기록됩니다. 별표를 누르면 고정되고, 저장된 쿼리는 **저장됨** 에서도 확인할 수 있습니다.

## 7. 저장 · 스케줄 · 대시보드

- **저장됨** — 저장된 쿼리를 폴더로 그룹화, 같은 테넌트 내 공유, CSV 내보내기
- **스케줄러** — 저장된 쿼리를 cron 일정으로 실행, 결과를 이메일/웹훅으로 전달 *(예정)*
- **대시보드** — 여러 차트 위젯을 하나의 뷰로 조립, 라이브 SQL 쿼리를 임베드해 자동 새로고침
- **차트** — 단일 쿼리에 고정된 차트 빌더
- **리포트** — *(예정)*

## 8. AI 어시스턴트 패널 활용

오른쪽 상단 AI 버튼을 누르면 채팅 패널이 슬라이드로 열립니다. 패널은 **현재 SQL**, **활성 연결**, **방언**, **선택된 스키마 스니펫** 을 자동으로 컨텍스트로 받습니다.

자주 쓰는 프롬프트:

- *"이 쿼리를 최적화해줘"* — 인덱스/CTE를 활용한 재작성 버전 제안
- *"이걸 한국어로 풀어서 설명해줘"* — 줄별 설명
- *"이 쿼리에 뭐가 잘못됐어?"* — 실제 스키마 기반 디버깅
- *"가장 최근 주문의 데이터 보여줘"* — 인라인으로 NL→SQL 변환

채팅에 표시되는 코드 블록에는 **복사** 와 **적용** 버튼이 있습니다. 적용 버튼은 SQL을 워크스페이스 에디터로 바로 보냅니다.

`⌘I` (Linux는 `Ctrl+I`) 로 어디서나 패널을 토글할 수 있고, `Esc` 로 닫힙니다.

## 9. 관리자 기능

관리자 사용자(첫 가입자 + 기존 관리자가 승격한 사용자)는 추가 메뉴를 사용할 수 있습니다:

- **사용자 관리** — 사용자 목록, 역할 변경, 비활성화
- **AI 프로바이더** — 테넌트가 사용할 Claude/OpenAI/Google 키 관리
- **AI 컨텍스트** — 시스템 프롬프트 커스터마이즈, 용어집 업로드, 테이블 설명 입력
- **상태·에러** — 최근 앱 오류와 스택 트레이스(서버 측 필터링됨)
- **감사 로그** — 모든 로그인, 쿼리 실행, 역할 변경 기록

## 10. 트러블슈팅

**`/home` 에서 "Failed to fetch"**
- 브라우저가 API에 도달하지 못한 상태. `docker compose logs -f web` 으로 컨테이너 헬스 확인. 보통 부팅이 아직 안 끝났을 가능성

**쿼리가 30초 넘게 걸리며 타임아웃**
- vibeSQL의 쿼리당 타임아웃은 30초입니다. 에디터에서 `LIMIT` 또는 필터를 추가하고 다시 실행하세요

**"CONNECTION_ENCRYPTION_KEY must be set"**
- env 파일에 암호화 키가 없습니다. `openssl rand -hex 32` 로 생성한 값을 `.env.docker` (또는 `.env.local`) 에 설정하고 재시작

**재시작 후 로그인 안 됨**
- `DATABASE_URL` 없이(인메모리 모드) 실행 중이면 계정이 재시작 시 사라집니다. 영속성을 원하면 `DATABASE_URL` 설정

**모든 쿼리에서 신뢰도가 "낮음"**
- AI가 충분한 스키마 컨텍스트를 받지 못한 상태. 스키마 페이지에서 활성 연결 introspection이 정상인지 확인. 관리자라면 AI 컨텍스트 메뉴에서 테이블 설명을 추가해보세요

**AI 어시스턴트 버튼이 페이지 액션과 겹침**
- v0.2에서 수정됨 — TopBar가 우측 164px를 예약합니다. 겹침이 보이면 옛 빌드 — 최신을 받으세요
